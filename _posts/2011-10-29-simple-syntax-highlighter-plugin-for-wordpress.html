I usually end up having a need for some sort of syntax highlighter on every blog I have ever contributed to. This goes double for my own blog. There are a ton of options out there, and it can make your head spin trying to decide what languages, features, short codes, and chrome spinners you want.

I decided it would be better to just have a small set of functionality from a well established tool. I have always been a fan of the syntax highlighter by Michael Samuel hosted on Google code.

This plugin is simple. It queues the scripts and style sheets, and it allows you to pick a theme. The usage follows the same rules as if you were using the highlighter any where else. See here for detailed instructions.

Simple Syntax Settings Menu

Easily switch between themes for the syntax highlighter. Ships with the four themes listed here. It is also easy to create your own themes. Just place the CSS file and preview thumb in the themes directory of the plugin.
